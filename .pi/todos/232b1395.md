# Goal
Add macOS Speech Recognition as an alternative backend to pi-listen, keeping all existing functionality.

## Status: ✅ COMPLETE

## Repository
`/Users/toenobu/ghq/github.com/codexstar69/pi-listen`

## Implementation Summary

### Files Changed
- `extensions/voice/config.ts` - Added `VoiceBackend` type and `backend` field
- `extensions/voice.ts` - Added Mac Speech session handling (~200 lines)
- `scripts/macspeech.swift` - Swift transcription binary (new)
- `scripts/macspeech` - Compiled binary (new)
- `README.md` - Updated documentation
- `docs/backends.md` - Complete backend documentation

### Features Added
1. **New backend option**: `"macspeech"` (local macOS Speech Recognition)
2. **Config setting**: `"backend": "deepgram" | "macspeech"` in settings.json
3. **63 languages supported** via macOS Speech framework
4. **Offline capable** after first use
5. **No API key required** for Mac Speech

### Usage
```json
// ~/.pi/agent/settings.json
{
  "voice": {
    "backend": "macspeech"  // or "deepgram" (default)
  }
}
```

### Commands Updated
- `/voice on` - Shows current backend
- `/voice info` - Shows backend info
- `/voice test` - Tests backend-specific requirements
